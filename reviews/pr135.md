## Issues Found

### Issue 1: `grub_errno` not set on authentication failure

**Location:** `grub-core/normal/auth.c`, `grub_auth_check_authentication`,
line ~288; same pattern in `password.c`, `password_argon2.c`,
`password_pbkdf2.c`

The patch replaces the `GRUB_ACCESS_DENIED` macro with bare
`GRUB_ERR_ACCESS_DENIED`. The macro expands to a `grub_error()` call
that sets `grub_errno` and `grub_errmsg`; the bare enum does neither.
The added `grub_puts_()` prints to the console but does not set error
state.

Callers in `menu.c` and `main.c` call `grub_print_error()` after
authentication failure, which checks `grub_errno` -- not the return
value. With `grub_errno` unset, `grub_print_error()` prints nothing
or a stale message from an earlier error.

A fix that achieves both goals (friendly message, correct error state):

```c
grub_errno = GRUB_ERR_ACCESS_DENIED;
grub_puts_ (N_("Authentication error"));
return GRUB_ERR_ACCESS_DENIED;
```
