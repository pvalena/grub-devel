## Additional findings

Per POSIX, `getline()` may allocate a buffer even when it
returns -1 (e.g., allocation succeeded but the read failed).
The old code set `line = NULL` at the top of each loop
iteration and broke on failure without freeing, so any
allocation on the failure path leaked. The fix uses `free()`
(not `grub_free()`), which is correct for host-side osdep
code that uses libc `getline()`.
