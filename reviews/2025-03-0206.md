# Review: MR !24 - Add convenience TARGET_CROSS

The implementation has a logic issue:

- `AC_CHECK_TOOL` automatically prepends `$ac_tool_prefix` (which is `$target_alias-`) to the tool name
- When `TARGET_CROSS` is set (e.g., to `aarch64-linux-gnu-`), the code passes `${TARGET_CROSS}objcopy` to `AC_CHECK_TOOL`
- This causes a double-prefix lookup: `$target_alias-${TARGET_CROSS}objcopy` (e.g., `arm-eabi-aarch64-linux-gnu-objcopy`)
- The first lookup always fails, then falls back to `${TARGET_CROSS}objcopy` which succeeds

This works but is inefficient (unnecessary failed lookup). More importantly, the use case is unclear:
- This block only executes when `target_alias != host_alias` (i.e., cross-compiling)
- If `TARGET_CROSS` is meant to replace setting `target_alias`, it won't work (block won't execute)
- If `TARGET_CROSS` supplements `target_alias`, you get the double-prefix issue

When would `TARGET_CROSS` be used instead of just setting `--target` correctly?
