## Additional findings

The change converts a single `GRUB_FILE_TYPE_LOOPBACK` check into a switch
that also skips `GRUB_FILE_TYPE_FS_SEARCH`, `GRUB_FILE_TYPE_GET_SIZE`,
`GRUB_FILE_TYPE_CAT`, `GRUB_FILE_TYPE_HEXCAT`, and `GRUB_FILE_TYPE_CMP`.
All five types are already skipped by the secure boot verifier in
`grub-core/kern/efi/sb.c`, which skips these plus many more (crypto keys,
configs, themes, etc.). The TPM verifier appropriately adds only the
inspection-only subset — it still measures configs, themes, and other
non-inspection types, which is correct for measured boot.

The pre-existing NULL dereference risk in `grub_tpm_verify_string` (no
default case for the `grub_verify_string_type` switch, leaving `prefix`
NULL) is unchanged by this MR. That issue was reported in the review
of MR !151.
