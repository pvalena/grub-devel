## Additional findings

The `--all` deletion for addresses uses `next_inter` for safe
iteration while calling `grub_net_network_level_interface_unregister`
and freeing each interface. PERMANENT interfaces are skipped with a
warning. This follows the same pattern as the existing single-delete
path.

Route deletion with `--all` walks the list freeing each route, then
sets `grub_net_routes = NULL`. DNS deletion with `--all` calls
`grub_net_remove_dns_server(&dns_servers[0])` in a loop — this works
because that function swaps the removed entry with the last element
and decrements the count.

The `--set` option accepts `[NUMBER:]VARNAME` format. The index
parsing uses `grub_strtoul(varnames[i], &q, 10)` and verifies `q`
points to the `:` separator. If the index exceeds the number of
items, the variable is unset rather than left stale. This is a
reasonable design choice.

All six commands (3 del + 3 ls) are converted from `grub_command_t`
to `grub_extcmd_t`, with matching `register_extcmd`/`unregister_extcmd`
in GRUB_MOD_INIT/FINI.
