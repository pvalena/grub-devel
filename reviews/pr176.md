## Additional findings

Bulk import of libgcrypt 1.12.2 (178 files, ~24k lines) adding
ML-DSA (Dilithium) post-quantum signature verification. Four new
GRUB-specific patches (17-20) adapt the imported code: disable
unavailable libgpg-error functions (`log_fatal`, `gpgrt_*`,
`gpg_err_set_errno`), suppress warnings, disable the `hash_buffers`
fast-path (would crash — GRUB strips per-algorithm implementations),
and add a missing `_gcry_mpih_add_1_lli` declaration.

The most significant GRUB patch (`18_dilithium`) is a FIPS 204
compliance fix: upstream `dilithium_verify` skips the domain
separator entirely when no context is provided (`prelen = 0`), but
FIPS 204 requires the 2-byte prefix `(0, 0)` even for empty context.
Without this fix, GRUB would reject valid FIPS 204 ML-DSA signatures.

The import script (`import_gcry.py`) un-skips `_gcry_md_extract`
(needed for SHAKE XOF extraction by Dilithium) and adds skip entries
for `mldsa_generate`, `mldsa_sign`, `randombytes` (GRUB only
verifies). The MLDSA test module covers all three security levels
(44/65/87) with and without context strings.
