# AI Review: MR !38 - Use ET_DYN images and PHDRs

Changes GRUB image generation from ET_REL to ET_DYN to avoid complex LoongArch relocations. Uses `-shared` linking so linker handles relocations instead of grub-mkimage.

- **IA-64 trampoline removal** (util/grub-mkimagexx.c:1050): R_IA64_PCREL21B now errors "trampolines not supported". Assumes `-shared` linker handles these, but needs IA-64 testing to verify.

- **R_IA64_SEGREL64LSB handler removed**: Deleted from `relocate_addrs()`. If linker still emits this with `-shared`, will cause unhandled relocation error.

- **Relocation value assumptions** (R_IA64_DIR64LSB, R_IA64_FPTR64LSB, R_LARCH_64): Changed from `old_value + symbol + addend` to `symbol + addend`. Assumes initial value is 0 (true for ET_DYN, not ET_REL).

- **IA-64 function descriptor GP** (util/grub-mkimagexx.c:705): GP pointer changed from 0 to `layout->vaddr_diff + layout->dyn_pltoff`. Correctness unclear without IA-64 ABI expertise.

High-risk architectural change affecting all platforms. Needs thorough testing, especially IA-64 and LoongArch64.
