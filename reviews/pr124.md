### Issue 1: Memory leak in `set_loader_default_entry()` (bli.c)

**Location:** `grub-core/commands/bli.c`, `set_loader_default_entry()`

Neither `oneshot_entry` nor `default_entry` are freed after being passed
to `grub_env_set()`. Both are heap-allocated by `get_efivar()` via
`grub_malloc()`. Since `grub_env_set()` copies the value, these are
leaked on every call.

```c
static void
set_loader_default_entry (void)
{
  char* default_entry;
  char* oneshot_entry = get_efivar("LoaderEntryOneShot");

  if (oneshot_entry != NULL)
    {
      grub_env_set ("default", oneshot_entry);
      // oneshot_entry is never freed
      ...
    }
  else
    {
      default_entry = get_efivar("LoaderEntryDefault");
      if (default_entry != NULL)
        {
          grub_env_set ("default", default_entry);
          // default_entry is never freed
        }
    }
}
```

```diff
--- a/grub-core/commands/bli.c
+++ b/grub-core/commands/bli.c
@@ set_loader_default_entry

     {
       grub_env_set ("default", oneshot_entry);
+      grub_free (oneshot_entry);
       grub_efi_set_variable_to_string("LoaderEntryOneShot",
 				      &bli_vendor_guid, "", 0);
     }
   else
     {
       default_entry = get_efivar("LoaderEntryDefault");

 	{
 	  grub_env_set ("default", default_entry);
```

### Issue 2: Incorrect error check in `bump_counter()` (bls_bumpcounter.c)

**Location:** `grub-core/commands/bls_bumpcounter.c`, `bump_counter()`

if parsing succeeded. However, `grub_strtol()` (via `grub_strtoull()`)
**never** sets `*end` to NULL. On parse failure, it sets `grub_errno` to
`GRUB_ERR_BAD_NUMBER` and leaves `*end` pointing into the string. The
`tmp != NULL` check will always be true, so parse failures are silently
ignored, leading to `tries_left` being 0 and incorrect renaming.


### Issue 3: Mutation of `const` environment value in `get_entry_number()` (menu.c)

**Location:** `grub-core/normal/menu.c`, `get_entry_number()`

`grub_env_get()`, which returns a pointer to internal storage. The new
code modifies this memory via `*ext = '\0'`:

```c

...
val = grub_env_get (name);
...

if (ext && grub_strcmp (ext, ".conf") == 0)

    res = menuentry_eq (e->id, val);
    *ext = '.';        // restores it
```

obtained from `grub_env_get()` is undefined behavior and corrupts the

### Issue 4: Mutation of environment variable `blsdir` via cast (bls_bumpcounter.c)

**Location:** `grub-core/commands/bls_bumpcounter.c`,
`grub_cmd_bls_bumpcounter()`

The pointer returned by `grub_env_get("blsdir")` is cast away from const
and mutated by `use_efi_delimiter_for_path()`. This modifies the internal
environment storage, permanently changing `/` to `\` in the `blsdir`
environment variable for all subsequent users.

```c
blsdir = grub_env_get ("blsdir");
if (blsdir)
  {
    use_efi_delimiter_for_path ((char*) blsdir);  // mutates env storage
  }
```

Should use `grub_strdup()` before mutation.

### Issue 5: `handle` not closed on error paths (bls_bumpcounter.c)

**Location:** `grub-core/commands/bls_bumpcounter.c`,
`grub_cmd_bls_bumpcounter()`

After the file is successfully opened via `root->Open()`, the `finish`
label frees `file_info` but never calls `handle->Close(handle)`. On any
error path after the file is opened (e.g., `bump_counter()` fails,
filename too long, `GetInfo` fails), the file handle leaks.

```c
finish:
  grub_free(file_info);
  return GRUB_ERR_BUG;    // handle is not closed
```

### Issue 6: Incorrect include guard name (filesystem.h)

**Location:** `include/grub/efi/filesystem.h`

The header guard is `GRUB_EFI_TPM_HEADER` instead of something like
`GRUB_EFI_FILESYSTEM_HEADER`. This will conflict with the actual TPM
header at `include/grub/efi/tpm.h` which uses the same guard, causing
one of them to be silently skipped.

```c
#ifndef GRUB_EFI_TPM_HEADER
#define GRUB_EFI_TPM_HEADER 1
```

### Issue 7: `LoaderFeatures` uses platform-dependent `sizeof(long)` (bli.c)

**Location:** `grub-core/commands/bli.c`, `set_loader_features()`

The Boot Loader Interface specification defines `LoaderFeatures` as a
64-bit bitmask. Using `long` and `sizeof(long)` is platform-dependent
(4 bytes on 32-bit ia32 GRUB, 8 bytes on 64-bit). The `bli` module has
`enable = efi` which includes 32-bit platforms. This would write an
incorrectly-sized variable on i386-efi. Should use `grub_uint64_t`.

```c
static long loader_features = ...;
return grub_efi_set_variable("LoaderFeatures", &bli_vendor_guid,
                             &loader_features, sizeof(long));
```
