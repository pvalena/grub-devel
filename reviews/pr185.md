## Additional findings

The original code leaked S-expressions allocated by `_gcry_sexp_build()`
on early-return error paths (e.g., if the second build fails, the first
sexp is never released). The fix initializes all three sexp pointers to
NULL, uses goto for all error paths, and releases all at a single exit
point. `_gcry_sexp_release(NULL)` is safe per gcrypt convention. The
debug `_gcry_sexp_dump()` calls are also removed.
