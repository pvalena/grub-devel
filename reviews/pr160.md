## Issues Found

### Issue 1: Device not closed on mount failure in
btrfs-list-subvols

**Location:** `grub-core/fs/btrfs.c`,
`grub_cmd_btrfs_list_subvols()`, line ~2729

```c
data = grub_btrfs_mount(dev);
if (!data)
    return grub_error (...);  /* dev not closed */
```

`grub_device_open(devname)` succeeds, but `grub_btrfs_mount` fails
(non-btrfs device, corrupted superblock). The early `return` bypasses
the `out` label which calls `grub_device_close(dev)`. The device
handle leaks.

**Draft fix:**

```diff
--- a/grub-core/fs/btrfs.c
+++ b/grub-core/fs/btrfs.c
   data = grub_btrfs_mount(dev);
   if (!data)
-    return grub_error (GRUB_ERR_BAD_ARGUMENT, "could not open device");
+    {
+      grub_device_close (dev);
+      return grub_error (GRUB_ERR_BAD_ARGUMENT, "could not open device");
+    }
```

---

### Issue 2: `output` string leaked in btrfs-list-subvols

**Location:** `grub-core/fs/btrfs.c`,
`grub_cmd_btrfs_list_subvols()`, lines ~2805-2815

```c
if (output)
    grub_env_set(varname, output);

out:
  free_iterator(&desc);
  /* ... */
  return GRUB_ERR_NONE;
  /* output is never freed */
```

When `-o VAR` is used, `output` is built via `grub_xasprintf` in
the loop. After `grub_env_set` copies it into the environment, the
original allocation is never freed. Leaked on every invocation with
`-o`.

**Draft fix:**

```diff
--- a/grub-core/fs/btrfs.c
+++ b/grub-core/fs/btrfs.c
   if (output)
     grub_env_set(varname, output);
 
 out:
   free_iterator(&desc);
+  grub_free (output);
 
   if (data)
```
