# AI Review: MR !165 - net_add_addr PREFIX support

2 commits adding optional `/PREFIX` notation to `net_add_addr` ADDRESS
argument, plus documentation.

**Commits:**
1. **50b223175** - net: net_add_addr: adding ability to specify PREFIX
   with IP address
2. **af1a834d3** - docs: documenting setting network prefix with the
   command net_add_addr

## Issues Found

### Issue 1: /0 prefix triggers undefined behavior in mask shift

**Location:** `grub-core/net/net.c`, `grub_cmd_addaddr()`, line ~1182

```c
cidr = (int)grub_strtol(&p[1], &q, 10);
if ( (! q) || (q[0] != '\0') || (cidr > 31) || (cidr < 0) )
    return grub_error (..., N_("invalid network prefix"));
```

The validation accepts `cidr = 0` (e.g., `10.0.0.1/0`). This value
is passed to `grub_net_add_ipv4_local(inf, 0)`, which computes:

```c
ip_cpu & (0xffffffff << (32 - mask))  // mask=0 → 0xffffffff << 32
```

Left-shifting a 32-bit value by 32 is undefined behavior (C99
6.5.7p3). On x86 the shift count is masked to 5 bits (`<< 32`
becomes `<< 0`, result `0xffffffff`), while ARM's barrel shifter
returns 0 for shifts >= 32. The route base address is
architecture-dependent.

**Draft fix:**

```diff
--- a/grub-core/net/net.c
+++ b/grub-core/net/net.c
@@ -1182,7 +1182,7 @@
       cidr = (int)grub_strtol(&p[1], &q, 10);
-      if ( (! q) || (q[0] != '\0') || (cidr > 31) || (cidr < 0) )
+      if ( (! q) || (q[0] != '\0') || (cidr > 31) || (cidr < 1) )
         return grub_error (GRUB_ERR_BAD_ARGUMENT,
                            N_("invalid network prefix"));
```

## Additional findings

The `(! q)` check in the validation is dead code — `grub_strtol`
never sets `*endp` to NULL. On parse failure, `*endp` points into
the string and `grub_errno` is set. The actual parse validation
comes from `q[0] != '\0'`, which correctly catches non-numeric
input.
