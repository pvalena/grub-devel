## Additional findings

The old threshold (`ref_count > 1`) was correct when
`grub_dl_load_file` held a load-time ref through the module's
lifetime — but commit 5768812175 (2011) moved the unref into
`grub_dl_load_file` itself, dropping it after GRUB_MOD_INIT
returns. After that, autoloaded modules that don't self-ref
steady-state at ref_count == 0, which the `> 1` guard passes
through. The subsequent `grub_dl_unref` underflows from 0, and
commit 500e5fdd8 (2024) made that underflow fatal. The new
`> 0` threshold with the removed `grub_dl_unref` matches
`grub_dl_unload`'s own internal guard.

The regression test covers 4 cases: ref_count == 0 (gzio),
ref_count > 0 (normal), no-such-module, and dependency-held
(fshelp, verifying CVE-2020-25632's guarantee). The `flatten()`
helper for normalising serial console line-wrap is a nice touch.
