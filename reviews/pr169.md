## Additional findings

The `envblk.h` fix is correct — the header uses `grub_size_t` without
including its definition. All existing users happened to include
`types.h` first, masking the missing dependency.

The loadenv unit test (`loadenv_unit_test.c`) covers: open/validate
(valid and invalid signature), set (append, in-place overwrite with
longer and shorter values), escape handling (`\\` and `\n`), delete,
delete of non-existent key, out-of-space rejection, and entry counting.
The `grub_free(bad)` after a failed `grub_envblk_open` is correct —
`grub_envblk_open` does not take ownership on failure.

The FDT unit test (`fdt_unit_test.c`) uses a `grub_uint64_t` array
for the FDT buffer, guaranteeing the 8-byte alignment the memory
reservation block requires. Tests cover tree creation, header
validation, nested node insertion, property set/get/overwrite/grow,
node iteration (`first_node`/`next_node`), and error paths for
undersized buffers, non-FDT blobs, and misaligned offsets.

The image unit test (`image_unit_test.c`) uses a procfs-backed
in-memory file provider with a mutable name buffer. The same
`provider_entry` is registered once and the name is changed via
`grub_snprintf` before each decode — this works because GRUB procfs
stores the name pointer, not a copy. Pixel validation uses the
`mode_info` structure to decode RGB channels at their actual bit
positions, with a tolerance parameter for lossy JPEG. Malformed inputs
(truncated PNG, bad TGA bit depth, wrong JPEG SOI marker) are tested
to ensure graceful failure.

The end-to-end test (`grub_cmd_save_env.in`) creates a real ext2
disk image, stages a grubenv, runs GRUB under QEMU, and verifies
results both inside GRUB (reload after save) and on the host
(`grub-fstest` readback). Properly skips on non-x86 platforms and
when `mke2fs`/`debugfs` are unavailable (`exit 77`).
