## Issues Found

No issues found.

**Commit 1**: In `grub_cmd_nslookup` (dns.c:648), the stack-allocated
`cmd_server` struct had its `.option` field uninitialized when called
with two arguments. The fix sets it to `DNS_OPTION_PREFER_IPV4`,
matching the default behavior of `net_add_dns`.

**Commit 2**: Adds `http_fin` callback to `grub_net_tcp_open` (was
NULL). When a server returns an error (404/500) and closes the
connection, the FIN handler signals EOF immediately instead of
spinning for the full 30-second poll timeout. `http_fin` uses
`GRUB_NET_TCP_DISCARD` (orderly FIN) vs `http_err`'s
`GRUB_NET_TCP_ABORT` (RST), which is correct for a server-initiated
graceful close.

**Commit 3**: Adds 11 test scripts with Python mock servers (DHCP,
DNS, TFTP, HTTP, IP fragmentation). Test infrastructure runs under
QEMU with tap networking. The mock servers are minimal,
purpose-built, and test-only.
