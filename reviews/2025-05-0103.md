# AI Review: MR !42 - Add xHCI support

Adds USB 3.0 (xHCI) controller driver. 2963 lines based on SeaBIOS implementation, tested on QEMU and MSC C6B-CFLR boards with USB mass storage, DVD burner, and hubs.

- **Potential double-free** (grub-core/bus/usb/xhci.c:2099,2196): `grub_xhci_check_transfer()` frees `transfer->controller_data` (line 2099) without setting it to NULL. If `grub_xhci_cancel_transfer()` is subsequently called on the same transfer, it retrieves the dangling pointer (line 2142-2143) and frees it again (line 2196), causing double-free. Should set `transfer->controller_data = NULL;` after line 2099.

- **Incomplete module cleanup** (xhci.c:2574-2578): `GRUB_MOD_FINI` halts/resets controllers but doesn't free DMA allocations (`devs_dma`, `eseg_dma`, `cmds_dma`, `evts_dma`, `spba_dma`, `spad_dma`) or global `xhci` controller list. Module unload leaks DMA memory.

**Note**: At 2963 lines, exhaustive review is impractical. Focused on resource management and integration points. Needs extensive hardware testing beyond QEMU, including regression testing on OHCI/UHCI/EHCI controllers.
