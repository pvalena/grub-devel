# AI Review: MR !32 - Relax memory allocation type

**Critical: Conflicts with MR !29**
- MR !29 (2025-04-0139) modifies the same function with a two-tier allocation strategy
- These changes cannot coexist
- MR !29 is dated Apr 11, this MR is Apr 16

**Concern: Removes platform-specific memory limits**
- Changing to `GRUB_EFI_ALLOCATE_ANY_PAGES` makes the `GRUB_EFI_MAX_USABLE_ADDRESS` parameter ignored
- This defeats the purpose of platform-specific memory limits (e.g., 4GB limit for Snapdragon X Elite)
- Per EFI spec: "On input, the address pointed to by Memory is ignored" for AllocateAnyPages
- Could allocate memory above 4GB on platforms that crash when accessing >4GB addresses

**Alternative approach:**
- If running out of memory with AllocateMaxAddress, the two-tier strategy in MR !29 is better
- It tries <4GB first, then falls back to higher addresses only if needed
- This preserves compatibility while solving the 710MB kernel load issue
