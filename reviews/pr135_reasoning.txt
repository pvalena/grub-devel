== Re-review summary ==

The initial review found that replacing GRUB_ACCESS_DENIED (which calls
grub_error(), setting grub_errno) with bare GRUB_ERR_ACCESS_DENIED left
grub_errno unset. Callers in menu.c and main.c call grub_print_error()
which checks grub_errno, not the return value.

The updated commit introduces grub_user_error() in kern/err.c: a variant
of grub_error() that sets grub_errno and formats grub_errmsg WITHOUT the
__FILE__:__FUNCTION__:__LINE__: prefix. This cleanly solves both the
original complaint (unfriendly file:line in error messages) and the bug
(grub_errno not set).

All paths to the access_denied label in grub_auth_check_authentication
now call grub_user_error() before the goto:
- grub_username_get failure (line ~258)
- grub_password_get failure (line ~264)
- unknown user or no callback (line ~275)
- callback called but is_authenticated false -- grub_user_error was
  called inside check_password (password.c, argon2, pbkdf2)

The bare "return GRUB_ERR_ACCESS_DENIED" at the access_denied label
(line ~291) is correct because grub_errno was set by the grub_user_error
call earlier in the flow.

== Issue 1: Missing format attribute ==

Discovery:
Compared grub_user_error declaration with grub_error declaration in
include/grub/err.h. grub_error has:
  __attribute__ ((format (GNU_PRINTF, 5, 6)));
grub_user_error has no format attribute.

Source trace:
grub_user_error takes (grub_err_t n, const char *fmt, ...) where fmt
is argument 2 and varargs start at argument 3. The correct attribute
would be format(GNU_PRINTF, 2, 3).

Consequence:
Without the attribute, passing mismatched format arguments (e.g.,
grub_user_error(GRUB_ERR_X, "value: %d")) compiles without warning
but produces garbage output or crashes at runtime from va_arg type
mismatch. Current callers all use plain strings with no format args,
so there is no immediate bug, but the missing attribute is inconsistent
with grub_error and removes a safety net for future callers.

Fix assessment:
Adding __attribute__ ((format (GNU_PRINTF, 2, 3))) to the declaration
is a one-line change with no side effects. It matches the project's
existing pattern for grub_error.
