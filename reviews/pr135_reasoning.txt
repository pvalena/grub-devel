== Issue 1: grub_errno not set on authentication failure ==

Discovery:
The patch replaces GRUB_ACCESS_DENIED with GRUB_ERR_ACCESS_DENIED in
four files. Checked whether these are equivalent.

Source trace -- macro definition:
include/grub/crypto.h:607:
  #define GRUB_ACCESS_DENIED grub_error (GRUB_ERR_ACCESS_DENIED, N_("access denied"))

include/grub/err.h, grub_error macro:
  #define grub_error(n, fmt, ...) \
    grub_error (n, __FILE__, __FUNCTION__, __LINE__, fmt, ##__VA_ARGS__)

So GRUB_ACCESS_DENIED expands to the function call
  grub_error(GRUB_ERR_ACCESS_DENIED, __FILE__, __FUNCTION__, __LINE__,
             N_("access denied"))

Source trace -- grub_error implementation:
grub-core/kern/err.c:
  grub_error() does:
    1. grub_errno = n;
    2. grub_snprintf(grub_errmsg, ..., "%s:%s:%d:", file, function, line);
    3. grub_vsnprintf(grub_errmsg + m, ..., fmt, ap);
    4. return n;

So GRUB_ACCESS_DENIED: sets grub_errno, formats grub_errmsg with
file:function:line prefix, returns GRUB_ERR_ACCESS_DENIED.

Bare GRUB_ERR_ACCESS_DENIED: just the enum value (60). Does not call
grub_error(). Does not set grub_errno. Does not set grub_errmsg.

Source trace -- what the patch does in auth.c:
Old code (access_denied label, line ~285 on master):
  return GRUB_ACCESS_DENIED;
  // expands to grub_error(...) which sets grub_errno + grub_errmsg

New code (line ~288 on pr135):
  return GRUB_ERR_ACCESS_DENIED;
  // bare return, grub_errno unchanged

The patch also adds grub_puts_ calls to print user-friendly messages
to the console, but grub_puts_ does not interact with grub_errno or
grub_errmsg at all -- it is a direct console output function.

Source trace -- caller impact:
grub-core/normal/menu.c:216:
  err = grub_auth_check_authentication (entry->users);
  if (err)
    {
      grub_print_error ();
      grub_errno = GRUB_ERR_NONE;
      return;
    }

grub-core/normal/menu.c:919:
  err2 = grub_auth_check_authentication (NULL);
  if (err2)
    {
      grub_print_error ();
      grub_errno = GRUB_ERR_NONE;
      continue;
    }

grub-core/normal/main.c:458:
  err = grub_auth_check_authentication (NULL);
  // later: grub_print_error ();

All callers check the return value (err) for flow control, then call
grub_print_error() to display the error.

grub_print_error() implementation (kern/err.c):
  do {
    if (grub_errno != GRUB_ERR_NONE)
      grub_err_printf (_("error: %s.\n"), grub_errmsg);
  } while (grub_error_pop ());

grub_print_error() checks grub_errno, NOT the return value. If
grub_errno is GRUB_ERR_NONE (because the patch never set it),
grub_print_error() prints nothing. If grub_errno has a stale value
from a previous grub_error() call elsewhere, it prints the wrong
message.

Consequence:
On origin/master: authentication failure returns via GRUB_ACCESS_DENIED
→ grub_errno set → caller's grub_print_error() prints
"error: auth.c:grub_auth_check_authentication:285:access denied."
(the message the commit author correctly identifies as unfriendly).

On pr135: authentication failure returns via bare GRUB_ERR_ACCESS_DENIED
→ grub_errno NOT set → caller's grub_print_error() either prints
nothing (grub_errno is GRUB_ERR_NONE) or prints a stale unrelated
error. The user does see "Authentication error" from the grub_puts_
call earlier, but grub_errno is in an incorrect state.

This also creates an inconsistency within grub_auth_check_authentication
itself: the grub_is_cli_disabled() path (line ~226 on pr135) still
returns GRUB_ACCESS_DENIED (sets grub_errno), while the access_denied
label returns bare GRUB_ERR_ACCESS_DENIED (does not set grub_errno).

Note on check_password callbacks: the return value of callback in
password.c, password_argon2.c, password_pbkdf2.c is ignored by the
caller in auth.c (line ~274: `cur->callback(login, entered, cur->arg);`
with no assignment). The old GRUB_ACCESS_DENIED set grub_errno as a
side effect; the new bare return does not. Since the caller ignores
the return value, the direct impact of the change in the callbacks
is limited to grub_errno state. However, using bare GRUB_ERR_ACCESS_DENIED
in a grub_err_t-returning function without setting grub_errno is
inconsistent with the GRUB error handling convention (every function
returning non-GRUB_ERR_NONE should have called grub_error() first).

Fix assessment:
The commit author's motivation is valid: the old error message included
source file/function/line info because grub_error() embeds __FILE__,
__FUNCTION__, __LINE__ into grub_errmsg. This is not user-friendly.

The fix should set grub_errno without going through grub_error() (to
avoid the file:line prefix). The simplest approach:
  grub_errno = GRUB_ERR_ACCESS_DENIED;
  grub_puts_ (N_("Authentication error"));
  return GRUB_ERR_ACCESS_DENIED;

This sets grub_errno for callers, prints the user-friendly message
directly (bypassing grub_errmsg formatting), and returns the error
code. Callers' grub_print_error() would see grub_errno set but
grub_errmsg would be stale -- however, since grub_puts_ already
printed the message, the caller could skip grub_print_error() or
the stale message is acceptable. A cleaner approach would be to
also manually set grub_errmsg:
  grub_errno = GRUB_ERR_ACCESS_DENIED;
  grub_snprintf(grub_errmsg, sizeof(grub_errmsg), "%s",
                _("Authentication error"));
  return GRUB_ERR_ACCESS_DENIED;

This fully replaces what grub_error() would have done, but with a
user-friendly message and no file:line prefix.
