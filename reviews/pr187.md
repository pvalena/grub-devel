## Additional findings

The `ibm,arch-vec-5-platform-support` property is a byte array of
index/value pairs. The original code used
`grub_ieee1275_get_integer_property` with a cast to `(grub_uint32_t*)`
-- incorrect for byte-level data and problematic for strict aliasing
under clang. Replaced with `grub_ieee1275_get_property` which copies
raw bytes. The loop increment was split across the `for` statement
(`i++`) and the loop body (`i++`), totaling 2 per iteration. Now
consolidated into a single `i += 2` at the end of the body, with
the `for` increment clause left empty -- same semantics, clearer
intent.
