## Additional findings

The bracketed address fix (commit 1) changes `if (bracketed && *ptr == ']')
ptr++;` to an explicit check that returns 0 on missing `]`. Previously,
`[3fff::53` (no closing bracket) was silently accepted.

The zero compression checks (commits 3-4) work together: a `zerocompress`
bool tracks whether `::` has been seen. If `::` appears a second time,
the function rejects the input (`3fff::1::53`). Separately, after consuming
the second `:` in a `::` sequence, a new `*ptr == ':'` check rejects triple
colons (`3fff:::53`). These are distinct checks for distinct invalid patterns.

The superfluous compression check (commit 6) rejects addresses where `::` is
used but all 8 words are explicitly provided (`3fff:0:0:0::0:0:0:1`). The
condition `quaddot != -1 && word == 7` catches this: if zero compression was
used and the loop filled all 8 words, the compression is redundant.

The IPv4-mapped handler (commit 5) is the most complex. When the main loop
sees `.` after a parsed hex value, it enters the IPv4 path. The first octet
was already parsed as hex by the main loop (`grub_strtoul(ptr, &ptr, 16)`),
so a hex-to-decimal conversion loop reinterprets the hex digits as decimal
(`0x192` → digits 1,9,2 → 192). The remaining three octets are parsed
directly as decimal (base 10). The `t > 0x255` threshold for the first
octet is exactly right: `grub_strtoul("255", ..., 16) = 0x255`, so any
decimal value > 255 exceeds this hex representation. The two 16-bit words
are packed at `byte == 1` (octets 1-2 → `newip[word++]`) and after the loop
(octets 3-4 → `newip[word]`). The `word > 6` guard prevents array overflow
since the handler writes two consecutive words.

The hex-to-decimal conversion works correctly for all-decimal-digit inputs.
If hex letters (a-f) appear in the first octet (e.g., `::ab.2.3.4`), they
pass the `0x255` check and produce a deterministic but incorrect decimal
value (0xab → 111 instead of rejection). This is unlikely in practice since
hex letters are never valid in IPv4 octets, and the remaining three octets
are parsed as base 10, which naturally rejects hex letters.

The variable declaration move (commit 2) is purely cosmetic — `t` moves
from loop-local to function scope, and new variables (`byte`, `ip4`, `base`,
`zerocompress`) are added for use by later commits.
