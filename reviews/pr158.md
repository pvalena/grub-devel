## Additional findings

Commit 2 is more than a cosmetic warning fix. The original
code:

```c
grub_test_assert (err == GRUB_ERR_NONE
                  || GRUB_ERR_ACCESS_DENIED, ...);
```

evaluates as `err == GRUB_ERR_NONE || (18)` because
`GRUB_ERR_ACCESS_DENIED` is a non-zero constant.
The expression always evaluates to true, making the
assertion pass regardless of the actual error code.
The fix adds the missing `err ==` comparison:

```c
grub_test_assert (err == GRUB_ERR_NONE
                  || err == GRUB_ERR_ACCESS_DENIED, ...);
```

This appears at two locations in
`tests/appended_signature_test.c` (lines ~266 and ~277),
both corrected by this commit. With the fix, the test
properly verifies that the returned error is one of the
two expected values.
