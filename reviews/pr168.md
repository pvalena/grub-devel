## Additional findings

The platform abstraction is clean: `#if GRUB_MACHINE_EFI` wraps the
EFI variable path, `#elif __powerpc__ && GRUB_MACHINE_IEEE1275` wraps
the device tree path, and `#error` catches unsupported platforms. The
Makefile only enables the module for `efi` and `powerpc_ieee1275`,
so the `#error` is a safety net for future `enable` additions.

The `--efivar`/`-E` option is renamed to `--name`/`-N` (platform
neutral). The `DEFAULT_NAME` macro differs per platform:
`"GrubPcrSnapshot"` for EFI, `"grub,pcr-snapshot"` for ieee1275.
The 30-character limit on name length is preserved.

The `actual` output from `grub_ieee1275_set_property` is not checked
after the call. On OpenFirmware, `set-property` either fully succeeds
or fails (no partial writes), so checking `ret < 0` is sufficient.
