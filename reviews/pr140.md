# AI Review: MR !140 - Fix write_cr0 writing to CR4

1 commit fixing a copy-paste bug in the x86_64 EFI hardware features
module for libgcrypt acceleration.

**Commits:**
1. **5bea14dc4** - lib/hwfeatures-gcry: Fix write_cr0 writing to CR4

## Issues Found

No issues found.

`write_cr0()` contained `mov %0,%%cr4` instead of `mov %0,%%cr0` --
a copy-paste error from the adjacent `write_cr4()`. This meant
`enable_sse()` never actually cleared CR0.EM or set CR0.MP, and
`grub_reset_gcry_hwf_x86_64_efi()` never restored CR0 to its
original value. Both callers were silently corrupting CR4 with CR0
values instead. The fix is a single-character change (`cr4` → `cr0`
in the asm operand), correct and complete.
