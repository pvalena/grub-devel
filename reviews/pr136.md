# AI Review: MR !136 - lvm: allocate metadata buffer from raw contents

1 commit reducing LVM metadata buffer allocation from 2x the metadata
area (~2 MiB) to only the active raw metadata (~1 KiB typical).

**Commits:**
1. **14c756a0f** - lvm: allocate metadata buffer from raw contents

## Issues Found

No issues found.

The old code allocated `grub_calloc(2, mda_size)` and read the entire
metadata area, then handled circular wrap with a memcpy. The new code
reads just the MDA header into a stack buffer, extracts the raw_locn
offset/size, validates bounds thoroughly (overflow-safe addition via
`grub_add`, six-way bounds check on offset/size/header relationships),
then allocates and reads only the raw metadata with proper circular
wrap handling via two targeted disk reads. Read all validation paths,
the circular buffer arithmetic, and the NUL-termination (`metadatabuf
[mda_raw_size] = '\0'`). The `mda_raw_size > GRUB_SIZE_MAX - 1` check
prevents overflow in the `mda_raw_size + 1` malloc. All `goto fail`
vs `goto fail2` targets match whether `metadatabuf` has been allocated.
