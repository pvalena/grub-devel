COMMIT MESSAGE
--------------------------------------------------------------------------------

we encountered some issues with grub boot failure on Linux systems. Upon
investigation, it was found that the ext4 fs of the system was abnormal,
causing the ext2 module of GRUB to read empty file contents. Therefore,
a verification of file size was added
'''
error: ../../grub-core/loader/i386/pc/linux.c:170:invalid magic number.
error :
../../grub-core/loader/i386/pc/linux.c:418:you need to load the kernel
first

grub> hexdump /boot/vmlinuz-5.10.134-17.2.a18.x86_64
grub>
grub> cat /boot/vmlinuz-5.10.134-17.2 a18 x86_64
grub>
'''
Verifying the file size in advance may make it easier to identify
the true cause of the problem
